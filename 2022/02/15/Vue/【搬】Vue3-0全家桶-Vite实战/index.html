2020 年 09 月 18 号发布 Vue3 正式版 V3.0.0，命名为 One Piece

一、Vue3.0 新特性，Compiler 原理介绍
与 Vue2.x 相比，mount 50%提升，内存占用小 120%
核心代码+Composition API ：13.5kb，最小 11.75kb
所有 Runtime：22.5kb（Vue2 是 32kb）

为什么有这么大的性能提升？
优化案例 1：

  • 静态 Node 不再作更新处理（hoistStatic -> SSR 优化）
  • 静态绑定的 class，id 不再作更新处理
  • 在 mount 时，template 会编译成 AST 语法树，结合打包标记 PatchFlag，进行更新分析（动态绑定）
    优化案例 2:
  • 事件监听器 Cache 缓存处理（cacheHandlers）
  • hoistStatic 自动针对多静态节点进行优化，输出字符串

编译尝试地址：https://vue-next-template-explorer.netlify.app/

Vue3.0 带来的变化

  • 性能提升 1.3 ～ 2x
  • TS 支持，新增：Fragment、Teleport、Suspense
  • 按需加载（配合 vite）&组合 API

新增功能：

  • Fragment–不受根节点限制，渲染函数可接收 Array
  • Teleport–类似 Portal，随用随取，e.g.弹窗，Actions
  • Suspense–嵌套的异步依赖，e.g.async setu()

一些资料：
中文文档：https://v3.cn.vuejs.org
组合式 API：https://composition-api.vuejs.org/api.html
官方文档：https://v3.vuejs.org

为什么要用 Composition API？

  • Vue2 对于复杂逻辑组件，在后期变得无法维护
  • Vue2 中代码复用方法，如：Mixin，Filters 都有缺陷
  • Vue2 对 TS 支持不充分

Vue2 逻辑复用方式

  • Mixin（命名空间冲突、逻辑不清晰、不易复用）
  • scoped slot 作用域插槽（配置项多、代码分裂、性能差）
  • Vue2 对 TS 支持不充分

Composition API 相对与 Options API 而言：

  • 复杂组件逻辑进行分离（组合式 API+函数式编程）
  • 组件间逻辑共享
  • TS 支持

二、Vue3 全家桶实战（Composition API）
三、就项目升级方案+Vite 开发新姿势
Vite 概念
Vite（快的意思）是一个 HTTP 服务器，特殊的地方： 1.可以在单文件中书写 ES6 语法 2.支持热更新（请求的内容会被打包/更新）
3.Rollup 打包

Vite 项目地址
github.com/vitejs/vite

更新建议

  • 小规模尝试，小项目尝试
  • Vue2.x 项目如果已经稳定，不推荐更新
  • Vue3 专用迁移版本，对 Vue2 进行兼容
