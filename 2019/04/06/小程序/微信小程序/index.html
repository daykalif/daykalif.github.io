1、设计模式

1.1、SPA（页面设计思想）

  • 单页面 Web 应用（single page web application，SPA）

    • 所谓的单页面应用程序，实际上是指，用户通过浏览器访问一个网站的时候，网站中的内容切换，不会产生新的页面。
    • 在开发时，用户可访问的页面只有一个（主体页面），开发时为了适应开发者习惯，引入了一种新的开发模式，页面模板

  • MVVM 框架

    • model view view-model：模型视图视图模型
    • 实际上省略开发时对于页面 DOM 的操作，但是可以实现页面数据变化 内存数据也变化，内层数据变换页面数据也变换的功能

2、微信小程序构成

2.1、文件构成

  • js 文件：为微信小程序添加页面功能（必须）

  • json 文件：为微信小程序添加初始设置项（app 页面必须）

  • wxml 文件：微信标记语言（符合微信规范的标记语言）（必须）

    • html：超文本标记语言（符合 w3c 规范标签）

  • wxss 文件：微信样式表（可以被微信识别的 css 样式）

    • 在微信中 有些样式不一定是 HTML 和 CSS 默认显示效果
    • css：层叠样式表

2.2、程序结构构成

2.3、小程序的技术块

  • 框架：用于解释说明，程序运行时的一些基础构成

    小程序开发框架的目标是通过尽可能简单、高效的方式让开发者可以在微信中开发具有原生 APP 体验的服务。

    框架提供了自己的视图层描述语言 WXML 和 WXSS，以及基于 JavaScript 的逻辑层框架，并在视图层与逻辑层间提供了数据传输和事件系统，让开发者能够专注于数据与逻辑。

    框架的核心是一个响应的数据绑定系统。

    整个小程序框架系统分为两部分：视图层（View）和逻辑层（App Service）

    框架可以让数据与视图非常简单地保持同步。当做数据修改的时候，只需要在逻辑层修改数据，视图层就会做相应的更新。

  • 组件：对应 wxml，微信定义在程序中可以使用的内置标签

    • view：相当于 HTML 布局标签 div
    • text：相当于 HTML 布局标签 span

  • API：对应微信提供一些功能

3、APP 的构成

  • 在微信一个页面应用 .js , .json, .wxml, [.wxss]
  • 在微信项目中 app.wxml（容器）文件是真实存在的 （不对开发者开放，用户访问小程序直接访问该页面）

3.1、app.json

  • app.json 文件会引导小程序的启动，定义了模板页面的路径，导航窗口的表现形式
  • 该文件的默认结构
  • app.json 的配置项
属性 类型 必填 描述 最低版本
pages String Array 是 页面路径列表（定义小程序可用的模板文件）
window Object 否 全局的默认窗口表现（定义头部导航的显示效果）
tabBar Object 否 底部 tab 栏的表现
networkTimeout Object 否 网络超时时间
debug Boolean 否 是否开启 debug 模式，默认关闭
functionalPages Boolean 否 是否启用插件功能页，默认关闭 2.1.0

workers String 否 Worker 代码放置的目录 1.9.90
requiredBackgroundModes String Array 否 需要在后台使用的能力，如「音乐播放」
plugins Object 否 使用到的插件 1.9.6
preloadRule Object 否 分包预下载规则 2.3.0
resizable Boolean 否 iPad 小程序是否支持屏幕旋转，默认关闭 2.3.0
navigateToMiniProgramAppIdList String Array 否 需要跳转的小程序列表，详见 wx.navigateToMiniProgram 2.4.0

3.2、app.wxss 文件

  • 全局样式表：会被所有模板页面进行继承（模板页面中也会生效）

3.3、app.js 文件

  • 多模板页数据共享

4、Page 的构成

4.1、page.json

  • 单独对一个模板页面的展示进行设置

    • page.json 文件只能设置 app.json 文件中的 window 所包含的属性==>这些属性作为一级属性存在
    • page.json 文件中设置的属性优先级高于 app.json 中 window 优先级==>page.json 设置的属性会覆盖 app.json 的设置
    • page.json 文件在配置项上，删除了navigationStyle，增加了disableScroll

  • disableScroll 属性的作用==>overflow:hidden; (禁止模板页的上下滚动)

4.2、page.wxml

  • 功能和 HTML 文件一样
  • 只能使用微信提供的标签

4.3、page.wxss

  • 局部样式表：只对当前页面生效
  • 功能和 css 文件一样（css 选择器和样式属性）
  • 部分 css 属性和选择器在 wxss 中不起作用
  • （微信的自定义单位）rpx（responsive pixel）: 可以根据屏幕宽度进行自适应。规定屏幕宽为 750rpx。如在 iPhone6 上，屏幕宽度为 375px，共有 750 个物理像素，则 750rpx = 375px = 750 物理像素，1rpx = 0.5px = 1 物理像素。
    • 在 wxss 中使用 background：url（只接受网络路径），本地图片路径无法生效==>使用图片的 base64 码进行替代

4.4、page.js

  • 用于定义该模板可以被微信小程序使用
  • 需要去传入一些和页面有关的参数页面逻辑代码

options 参数的定义:

  • 取值类型 Object=={key1:value,key2:value……}

    • key:API 定义的具有固定意义的名称
    • value：该名称所对应的参数（固定类型的数据）

  • data：模板页面的数据仓库 == 用于定义页面中的数据

  • 其他

    用于定义在页面中被标签绑定的处理方法

    事件绑定的事件名

    1.在微信小程序的事件绑定中，一定需要遵循小程序规范bind事件名=函数名

    2.事件绑定后，可在标签中使用data-参数名=参数值的方式进行事件绑定

    3.被绑定的函数，有且仅有一个参数，该参数为 event 事件

    4.页面中通过 data-传递的数据，被写在了 event.currentTarget.dataset 中

    5.自定义函数中，this 表示的是当前页面（模板）的对象

4.4.1、页面生命周期

  • 一段代码从读取开始到代码执行完成，这段时间叫做代码的生命周期
  • 一个程序从打开运行到关闭运行，这段时间叫做程序的生命周期
    • 钩子函数：在特定的位置进行函数的挂载和调用
      • 在生命周期的特定时间强制执行的函数（此图有误：onshow 应该要比 onready 早）
      • onshow 的执行时间点比 onReady 要早
      • onLoad,onReady 只会在页面装载时执行一次

5、组件（标签）

5.1、内置组件（内置标签）

  • <navigator>可以简单的理解为<a>,可以完成页面的切换，但是不会刷新或重载页面
    • url 用于指定需要切换到的模板（一定是定义在 app.json 中 pages 属性中的模板）

5.2、自定义组件（自定义标签）

  • 代码复用（让开发者在少些代码的情况下完成复杂页面）
  • 使用文件和代码的方式，完成自定义标签的加载和使用
  • 我们构建的标签具有一定的样式和结构
  • 组件构建的要求：
    • 被组件化的代码（页面代码），一定是项目中重复使用频率很高的代码
    • 被组件化的代码一定是最小单元

6、逻辑层代码

7、wxml 的数据操作功能

  • wxml 不仅可以定义页面结构，同时可以完成数据的获取

7.1、数据加载（数据绑定）

  • 数据绑定使用 Mustache 语法（双大括号）将变量包起来，可以作用于页面

  • 页面中可以直接在双大括号中定义变量值：语法和 js 变量值的语法一致

7.2、数据循环（列表循环）

  • 使用数据操作属性 == 一定要被定义在标签属性上的
    • 使用时需要配合 wx:key 进行定义（如果不定义会导致警告，可能会导致审核不通过）
    • 对于循环而言，内置两个变量 index，item
      • index 获取循环的下标（次数）可以通过 wx:for-index 进行名称修改
      • item 循环得到的当前对象 可以通过 wx:for-item 进行名称修改

  • 可以循环对象

    • JS 中对象的循环 建议使用for in，（foreach）循环 == （会将原型链中的属性也循环出来）
    • 只循环对象的属性
      • index：就是对象的 key
      • item：就是当前 key 所对应的值

  • 对数值的循环

    • index 从 0 开始
    • item 从 0 开始

  • 对于字符串的循环

    • index 是字符在字符串的下标
    • item 是当前循环字符

7.3、数据判断（条件渲染）

  • 使用数据判断需要依赖标签属性==一定要被定义在标签属性上的
    • 判断当前标签是否需要被写入==是否被渲染
    • 依然遵循 JS 的数据类型转换规则（""==>false 0==>false null==>false......）

7.4、辅助数据操作标签

  • <block>:页面中不被解析，且不具有任何样式
    • 为数据判断和数据循环提供标签支持

8、API

  • API 是提供开发者对于通过 JS 方式调用相关功能的接口（内置的操作方法）

    • api：Application Programming Interface
    • 设备硬件调用 API
    • 微信内置功能的调用

  • 在所有的微信程序的 js 文件中都可以直接调用

8.1、wx.navigateTo（OBJECT）

  • 实现页面通过 js 方法进行切换

  • OBJECT 参数说明

8.2、wx.request(OBJECT)

  • 就是通过请求方式，获取远端数据（使用方法类似于 jquery 的$.ajax）

  • 请求地址必须是小程序授权地址

  • 开发者自行配置，微信进行审核

  • 请求地址必须是 HTTPS://加密协议

8.3、消息交互窗口

  • wx.showToast（OBJECT）显示消息提示窗口
