Vue2.x 应用范围非常广，有大量项目需要维护、升级
Vue3 使用 Proxy 实现响应式，Proxy 存在浏览器兼容问题，且不能 polyfill

Object.definedProperty

Object.defineProperty 的一些缺点（Vue3.0 启用 Proxy）

1.深度监听，需要一次性递归到底，一次性计算量大 2.无法监听新增属性/删除属性（Vue.set Vue.delete） 3.无法原生监听数组，需要特殊处理

Proxy 实现响应式

Proxy 兼容性不好，且无法 polyfill

1.Proxy 基本使用

2.Reflect 作用：
和 Proxy 能力一一对应
规范化、标准化、函数式
替代掉 Object 上的工具函数

3.Proxy 实现响应式：
深度监听，性能更好
可监听 新增/删除 属性
可监听数组变化

总结

Proxy 能规避 Object.defineProperty 的问题
Proxy 无法兼容所有浏览器，无法 polyfill
