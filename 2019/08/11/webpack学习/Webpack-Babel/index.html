问：module chunk bundle 的区别？

  • module - 各个源码文件，webpack 中一切皆模块，只要可以引用的都是模块
  • chunk - 多模块合成的，如 entry 可以定义 chunk，import()可以定义 chunk，splitChunk 可以定义 chunk
  • bundle - 最终的输出文件，可以是多个文件，一个 chunk 对应一个 bundle

ES6 Module 和 Commonjs 区别：

2.babel-polyfill

什么是 Polyfill?–【补丁】

  • babel-polyfill 是 core-js 和 regenerator 的集合
  • core-js【集成了一些常用的补丁】和 regenerator【支持 generator 语法】
  • Babel7.4 之后弃用 babel-polyfill，推荐直接使用 core-js 和 regenerator

babel 特点：

babel 只会检测语法，使语法符合 ES5 的语法规范
babel 不会解析新的 api，如 Promise，includes 方法
babel 不处理模块化（webpack）

babel-polyfill 按需引入：

文件较大
只有一部分功能，无需全部引入
配置按需引入

babel-polyfill 的问题：

会污染全局环境
如果做一个独立的 web 系统，则无碍
如果做一个第三方 lib，则会有问题

3.babel-runtime

.babelrc 中配置 babel-runtime，重新取变量名，解决 babel-polyfill 的全局污染

问：loader 和 plugin 的区别？

loader 模块转换器，如 less->css
plugin 扩展插件，如 HtmlWebpackPlugin

loader 将浏览器看不懂的代码变成看得懂的代码（翻译能力）
plugin（插件）拥有别的能力。具有可插拔（需要时安装，不用就删）。强大且灵活

问：常见 loader 和 plugin 有哪些？

https://www.webpackjs.com/loaders/
https://www.webpackjs.com/plugins/

问：babel 和 webpack 的区别？

babel-JS 新语法编译工具，不关心模块化
webpack-打包构建工具，是多个 loader plugin 的集合

babel-polyfill 会污染全局
babel-runtime 不会污染全局
产出第三方 lib 要用 babel-runtime

如 Class 可以用 function 模拟
