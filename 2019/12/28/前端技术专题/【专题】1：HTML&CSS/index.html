HTML 知识点

1.如何理解 HTML 语义化？
  • 让人更容易读懂（增加代码可读性）
  • 让搜索引擎更容易读懂（SEO）
  • 去掉或样式丢失的时候能够让页面呈现清晰的结构

 

2.默认情况下哪些 HTML 标签是块级元素，哪些是内联元素？
  • 块级元素：display 等于 block 或者 table 的元素；有 div h1 h2 table ul ol p 等；
    特点：独占一行
  • 内联元素：display 等于 inline/inline-block 的元素；有 span img input button 等；
    特点：不会独占一行，挨个往后排，直到浏览器的边缘换行为止

二、定位
1.absolute 和 relative 分别依据什么定位？

relative:依据自身定位
absolute 依据最近一层的定位元素定位
定位元素：absolute relative fixed，如果没有找到则定位到 body

 

2.居中对齐有哪些实现方式？
水平居中

1.inline 元素：text-align:center
2.block 元素：margin:auto
3.absolute 元素：left:50% + margin-left 负值（必须知道子元素的宽度）

 

垂直居中

1.inline 元素：line-height 的值等于 height 值
2.absolute 元素：top:50% + margin-top 负值（必须知道子元素的高度）
3.absolute 元素：transform(-50%,-50%)（不需要知道子元素的宽高）[CSS3 的属性，低版本浏览器不兼容]
4.absolute 元素：top,left,bottom,right = 0 + margin:auto（不需要知道子元素的宽高）[低版本浏览器也兼容]

 

三、图文样式

1.line-height 的继承问题？

写具体数值，如 50px，则继承该值
写比例，如 1.5，则继承该比例
写百分比，如 200%，则继承计算出来的值

 

四、响应式

1.rem 是什么？
  • px，绝对长度单位，最常用
  • em，相对长度单位，相对于父元素，不常用
  • rem，相对长度单位，相对于根元素，常用于响应式布局

 

2.如何实现响应式？
  • media-query（CSS3）,根据不同的屏幕宽度设置根元素 font-size
  • rem，基于根元素的相对单位

 

3.vw/vh 是什么？

rem 的弊端：“阶梯”性

网页视口尺寸：
window.screen.height    //屏幕高度（手机亮的屏幕高度）
window.innerHeight    //网页视口高度（浏览器去掉头，去掉下巴，剩下的中间视图高度）
document.body.clientHeight    //body 高度（网页呈现页面的高度）

vw/vh：
vh：网页视口高度的 1/100
vw：网页视图宽度的 1/100
vmax 取两者最大值，vmin 取两者最小值
